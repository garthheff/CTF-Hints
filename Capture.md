# Capture!

SecureSolaCoders has once again developed a web application. They were tired of hackers enumerating and exploiting their previous login form. They thought a Web Application (WAF) was too overkill and unnecessary, so they developed their own rate limiter and modified the code slightly.

Before we start, download the required files by pressing the Download Task Files button.

---

<details>
<summary>hint 1 – start by testing manually</summary>

Capture a few manual login attempts.

Try different bad logins and compare the responses carefully.

What changes between them?

</details>

---

<details>
<summary>hint 2 – review what the page gives you</summary>

Look closely at the login responses.

Is there anything in the returned page that helps you determine the username?

</details>

---

<details>
<summary>hint 3</summary>

If the application starts presenting a simple challenge, consider whether you really need to solve it by hand each time.

There may be an easy way to handle it with a python script.

</details>

---

<details>
<summary>hint 4 – use what you learned</summary>

Once you understand how to identify a valid username, repeat the same overall approach for the next stage.

You do not need a completely different method.

</details>
